# Unpipelined Command Grids — vex vs Redis 8.0.3

[Back to Benchmarks](benchmarks.md)

Per-command throughput delta (positive = vex faster), unpipelined, across
vex worker counts (w) and client connections (c). AWS EKS `c5a.2xlarge`
(8 vCPU), Go compare-client, n=50,000 × 3 runs per cell, June 2026.

Notes: w=1 columns are populated only for SET (other commands' w=1 data was
lost to a cluster-side job deletion mid-run). Deltas under ±8% are within
run-to-run noise. HGETALL numbers are client-parse-bound — see the
"Big-reply commands" section in [benchmarks.md](benchmarks.md). HINCRBY's
huge deltas reflect an anomalous Redis weakness, not vex strength.
